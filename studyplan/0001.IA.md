# Dominar IA — Um caminho a partir do Brasil

> *"Kayleos pôde fugir de Achaea e virar Deus. Meu jogo é aqui."*

O bloqueio é real. O compute de fronteira está fora de alcance. A cadeia de produção é americana.  
Mas a ideia que muda tudo ainda não foi descoberta. E ela não precisa de permissão para existir.

---

## O objetivo

Entender IA de verdade — não como consumidor, não como usuário de API — e criar algo simples que seja tão poderoso quanto os sistemas existentes.

Simples e poderoso é exatamente o que o transformer foi em 2017.  
"Attention is All You Need" — oito páginas.

---

## Fase 1 — A matemática que sustenta tudo

Não do zero. O foco é: **derivadas parciais, gradientes, distribuições**.

**Revisar:**
- Álgebra linear: vetores, matrizes, decomposições (SVD, eigenvalues)
- Cálculo multivariável: gradientes, jacobiana, chain rule
- Probabilidade: distribuições, expectativa, entropia, KL divergence

**Recurso principal:**
- [Bishop — Pattern Recognition and Machine Learning](https://www.microsoft.com/en-us/research/uploads/prod/2006/01/Bishop-Pattern-Recognition-and-Machine-Learning-2006.pdf) — gratuito, denso, honesto

**Critério de saída desta fase:**  
Conseguir derivar backpropagation na mão, sem consultar nada.

---

## Fase 2 — Implementar um transformer do zero

Sem HuggingFace. Sem abstrações. Só código e matemática.

**Recurso principal:**
- [Andrej Karpathy — Let's build GPT from scratch](https://www.youtube.com/watch?v=kCc8FmEb1nY)

Implementar junto, linha por linha. Quando o modelo de brinquedo começar a aprender — algo muda.

**O que implementar:**
1. Tokenização simples
2. Embedding + positional encoding
3. Self-attention (scaled dot-product)
4. Multi-head attention
5. Feed-forward layer
6. Transformer block completo
7. Treinar num corpus pequeno (Shakespeare, código, poesia)

**Critério de saída:**  
Modelo gerando texto coerente, treinado por você, sem biblioteca de alto nível.

## Fase 3 — Os papers originais, na ordem histórica

Ler os papers depois de implementar o núcleo muda a leitura. Você vai entender *por que* funciona, não só *o que* diz.

| Ano | Paper | Por que importa |
|-----|-------|-----------------|
| 2017 | Attention is All You Need | A ideia original |
| 2018 | BERT | Pré-treino bidirecional |
| 2019 | GPT-2 | Escala e geração |
| 2020 | Scaling Laws | Por que tamanho importa |
| 2022 | Constitutional AI (Anthropic) | Alinhamento como processo |
| 2023 | Toolformer | Modelos usando ferramentas |
| 2024 | Chain-of-thought prompting | Raciocínio emergente |

**Recurso:** [arxiv.org](https://arxiv.org) — tudo gratuito.

## Fase 4 — Open source como campo de treino

Sem frontier compute, o caminho é:

- **Llama 3** (Meta) — roda em hardware modesto com quantização
- **Mistral / Mixtral** — eficiente, acessível
- **Qwen** (Alibaba) — multilingual, forte em código

**O que fazer com eles:**
- Fine-tuning com LoRA (parâmetros mínimos, resultado real)
- Experimentos de alinhamento pequenos
- Integração com sistemas externos (o que você já sabe fazer)

**Ferramenta:** [Ollama](https://ollama.ai) — roda modelos localmente, sem depender de API americana.

## O que não perseguir (por ora)

- Treinar modelos de fronteira — não é caminho individual, nem nacional
- Depender de API americana para o núcleo do trabalho — open source primeiro
- Comparar o que você tem com o que a Anthropic tem — comparação errada
